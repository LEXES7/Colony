Questions

Asked and answered.

Do I need an API key?

No. If you're signed into Claude Code on your machine, Colony uses that — your existing subscription. Prefer pay-per-token? Drop an ANTHROPIC_API_KEY in .env instead. Either way, your credentials never touch anything but Anthropic.

Is my code sent anywhere?

Only to Anthropic's API for the model calls you trigger, using your own account — the same as using Claude directly. Colony itself has no servers, no accounts and no telemetry. The dashboard is bound to 127.0.0.1 and protected by a local 256-bit token.

Can the agents break my machine?

They can't run shell commands, can't reach the network through tools, and can only write inside the one folder you assign to their team — with symlinks resolved and credential folders denied outright. Secret files like .env and keys are refused at the tool level.

What happens when I hit my usage limit mid-build?

The venture pauses and everything is kept: approvals, finished tasks, the board. When your limit resets, press Resume — the team continues exactly where it stopped. Server crashes and restarts are handled the same way.

How much does it cost?

Colony is free and MIT-licensed. You pay only for your own Claude usage, and the dashboard shows live per-agent token meters so nothing surprises you. Cheap models handle routine work by default.

Can I use it on my real projects?

Yes — that's the point. Register any folder inside your workspace root and it gets an expert agent. Put a team on it when you want changes made. Use git and every change the team makes is reviewable and revertible.
